# E2E テスト(Playwright)

showcase デモを対象にした E2E 雛形です。

```bash
pnpm exec playwright install   # 初回のみ(ブラウザ取得)
pnpm e2e                        # ヘッドレス実行(dev サーバは自動起動)
pnpm e2e:ui                     # UI モードで対話実行
```

`playwright.config.ts` の `webServer` が showcase を自動起動します。
`baseURL` は `E2E_BASE_URL` で上書き可能(ステージング環境の検証など)。
`e2e/*.spec.ts` を追加してページ・フォーム・遷移を検証してください。
Cursor では Playwright 拡張(推奨拡張に含む)からテスト実行・トレース確認ができます。

## 業務の流れを通す E2E（business-flow.spec.ts）

画面ごとの確認とは別に、**一連の業務が最後まで通るか**を見ます。

| 見ていること | なぜ |
|---|---|
| 勤怠の集計が、そのまま給与の金額になる | ここが繋がっていないと給与計算が始まらない |
| 承認は**押した人の権限**を確かめる | 確かめないとチャンネルにいる人なら誰でも承認できる |
| マスタは**被参照**が見える | 使われている行を消すと業務が止まる |
| 主要画面へトップから辿れる | URL を知らない人が到達できるか |
| 存在しない URL は 404 | 白い画面にしない |

### 書き方の方針

**画面の細かい見た目は見ません。** そこは変わりやすく、壊れると
「E2E がまた落ちた」で無視されるようになります。見るのは**業務として成立しているか**だけです。

型検査も smoke も「画面が繋がっているか」は見ていません。**そこがこの E2E の担当**です。
