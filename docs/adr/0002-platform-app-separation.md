# ADR 0002: 基盤(packages)とアプリ(apps)の分離

- 状態: 承認
- 日付: 2026-07-09

## 背景
共通機能をアプリ内に散在させると重複・表記ゆれ・属人化が進む。一方、基盤を
アプリから自由に編集できると「共通部品」の意味が崩れる。

## 決定
- 共通機能は `packages/@platform/*` にラッパーとして集約し、公開 API のみ露出する。
- アプリは基盤の公開 API を使うだけで、`packages/` のソースは編集しない。
- 境界は ESLint(boundaries)・CODEOWNERS・CI・CLAUDE.md で多層に強制する。

## 結果
- ライブラリ差し替えが基盤内部で完結し、アプリは無変更。
- 基盤変更にレビュー負荷がかかる(意図的な摩擦。品質の担保として許容する)。
