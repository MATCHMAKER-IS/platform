# 3. 信頼性・観測性プリミティブを内製の依存ゼロパッケージで提供する

日付: 2026-07-10

## ステータス
承認

## 背景
社内アプリは Zoho をはじめ外部 API 連携が多い。障害時のリトライ暴発・二重実行・
障害伝播・送信取りこぼしが品質リスクになる。トレース/メトリクスも無く調査が困難だった。

## 決定
`@platform/observability` に、依存ゼロで以下を提供する:
- **トレーシング**(W3C traceparent 互換)・**メトリクス**(Prometheus 出力)
- **冪等性**(`withIdempotency`・アトミック予約で二重実行防止)
- **サーキットブレーカー**(`createCircuitBreaker`・障害伝播の遮断)
- **Outbox**(`relayOutbox`・DB コミットと外部送信の整合)
- **ヘルスチェック**(liveness/readiness)

ストアは抽象化し、実体(DB/Redis)はアプリが実装する。OpenTelemetry SDK 等の
重い依存は入れず、必要になれば exporter を差し替える方針。

## 結果
- 外部連携の信頼性が宣言的に確保できる(リトライ/冪等/遮断/確実送信)。
- 依存ゼロのため導入コストとサプライチェーンリスクが低い。
- 反面、OTel エコシステムの高機能(自動計装等)は将来課題として残る。
