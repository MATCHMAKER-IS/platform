# 0010: AI 呼び出しは AI Gateway(@platform/ai)経由を必須化

- 日付: 2026-07-14 / 状態: 採用

## 文脈
アプリが AI API を直接呼ぶと、ログ・コスト・トークン制御・キー管理・モデル切替が分散し統制不能になる(壁打ちの中核指摘)。

## 決定
アプリからの AI 呼び出しは **@platform/ai の Gateway 経由のみ**とする。Gateway がルーティング(モデル→プロバイダ)・トークン上限/予算・コスト計算・マスク付きログ・フォールバックを一元化。プロバイダ API 直叩きと API キー直書きは禁止(CLAUDE.md 開発ルール)。

## 検討した代替案と見送り理由
- 各アプリで SDK 直利用: 初速は出るが、コスト可視化と禁止事項の担保が不可能。
- 外部 LLM ゲートウェイ製品: 依存とコスト増。要件(ログ/予算/切替)は薄い自作で足りる。

## 影響
プロバイダ追加は `AiProvider` 実装のみ。実 API 疎通はキー設定後に各環境で確認(fetch 注入によりロジックはスモーク検証済)。
