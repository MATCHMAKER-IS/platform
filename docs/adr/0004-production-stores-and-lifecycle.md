# 4. 本番ストア実装・graceful shutdown・シークレット管理

日付: 2026-07-10

## ステータス
承認

## 背景
Outbox・分散ロック・冪等性・dedup がメモリ実装のみで、複数インスタンス/再起動に耐えられなかった。
また SIGTERM 時の後始末が無く、シークレットは環境変数平文前提だった(本番運用の穴)。

## 決定
- **本番ストア**: 抽象インターフェース(LockStore/SeenStore/IdempotencyStore/OutboxStore)に対し、
  Redis 実装(所有者照合による安全解放)と DB(SQL)実装を追加。各パッケージは依存を増やさないため
  クライアントを注入する形にし、フェイクで検証可能にした。
- **Graceful shutdown**: `@platform/core` に `createLifecycle` を追加。SIGTERM/SIGINT で
  登録の逆順に後始末(受付停止→完了待ち→接続切断)、タイムアウトで打ち切り。
- **シークレット**: `@platform/secrets` を新設。取得元を抽象化しローテーションに追随。
  `crypto.deriveKey` は共有既定 salt を廃止し必須化。

## 結果
- 複数インスタンス・再起動に耐える本番構成が可能に。ゼロダウンタイムデプロイに対応。
- シークレットの取得元差し替え・ローテーションが仕組み化。
- 反面、実際の Redis/DB 実装はアプリ側の配線(クライアント提供)が必要。
