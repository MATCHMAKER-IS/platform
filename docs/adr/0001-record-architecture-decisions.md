# ADR 0001: アーキテクチャ決定を記録する

- 状態: 承認
- 日付: 2026-07-09

## 背景
脱属人化の本質は「コードの読みやすさ」だけでなく「なぜそう決めたか」の保存にある。
決定の理由が失われると、後任は同じ議論を繰り返すか、意図に反した変更をしてしまう。

## 決定
重要な技術的決定は ADR(Architecture Decision Record)として `docs/adr/` に残す。
1 決定 = 1 ファイル。背景・決定・結果(トレードオフ)を簡潔に書く。

## 結果
- 決定の経緯が追える。新規参画者のオンボーディングが速くなる。
- ADR を書く手間が発生する(重要な決定のみに限定して軽量に運用する)。
