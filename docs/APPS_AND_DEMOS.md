# アプリとデモの紹介

このリポジトリには **実運用向けアプリ（apps/）** と **基盤の使い方を示すデモ（demos/）** が入っています。どちらも同じ基盤パッケージ（`packages/`）の上に載っています。

---

## ポート一覧

`pnpm dev` で**全アプリを一斉起動**できます（ポートは重複しないよう固定済み。`node tools/check-ports.mjs` で検査）。

| ポート | アプリ | 起動コマンド（個別） |
|---|---|---|
| 3000 | internal-app（社内アプリ） | `pnpm dev:internal` |
| 3001 | showcase（基盤デモ） | `pnpm dev:demos` |
| 3002 | crud-template（雛形） | `pnpm dev:crud` |
| 3003 | equipment-app（備品管理） | `pnpm dev:equipment` |
| 3004 | public-site（公開サイト） | `pnpm dev:site` |
| 3005 | platform-portal（基盤ポータル） | `pnpm dev:portal` |

ポートは各アプリの `package.json` の `dev` スクリプトで固定しています（`next dev --port XXXX`）。変更したらドキュメントも更新してください（不一致は `check-ports` が検出します）。

## apps/ — 実運用するアプリ（5つ）

### 1. internal-app — 社内アプリ本体
`pnpm dev:internal` → http://localhost:3000

**画面 79 / API 213 / DB モデル 65** の最大のアプリ。社内業務のほぼ全てがここに集約されています。

| 領域 | できること |
|---|---|
| 勤怠・給与 | 打刻、月次集計、給与計算、給与明細（PDF）、仕訳連携 |
| 経費・購買 | 経費申請、多段承認（金額ルーティング）、発注、検収、支払 |
| 会計 | 仕訳、月次決算、消費税（税率別・インボイス）、freee 連携 |
| 販売・請求 | 見積、受注、請求書（PDF）、入金消込 |
| 在庫・備品 | 在庫管理、棚卸、備品貸出 |
| 人事 | ユーザー・権限管理、e-ラーニング、アンケート |
| AI | AI ゲートウェイ（要約・生成）、RAG 文書検索（権限継承）、画像生成、チャットボット |
| 情シス | RPA、DB ビューア、監査ログ、通知センター、CMS、メンテナンスモード |
| 管理 | テーマギャラリー、辞書管理、設定確認、機能フラグ、自動化 |

**特徴的な機能**:
- **RAG 文書検索** — 社内文書を検索。**ユーザーの権限を継承**するので、見えない文書は結果に出ません
- **辞書機構** — 音声書き起こしの誤変換（例:「議事六」→「議事録」）を自動補正。CSV で一括登録でき、変更履歴も残ります
- **テーマギャラリー** — 11 スキンの切り替え、自社ブランドテーマの作成（色を選ぶだけ）、組織デフォルトの設定
- **設定の確認** — 今どの環境変数で動いているかを一覧（秘密値はマスク）

### 2. public-site — 公開サイト
`pnpm dev:site` → http://localhost:3004

**画面 10**。社外向けのコーポレートサイト。Tailwind + SEO 対応。

- ブログ（CMS 連携・下書きプレビュー）、お客様の声、お問い合わせフォーム
- 問い合わせは internal-app の受付 API へ転送され、社内の受信一覧に集約されます
- 唯一 SEO を効かせるアプリ（社内アプリは検索避け）

### 3. crud-template — 新規アプリの雛形
`pnpm dev:crud` → http://localhost:3002

**画面 79 / API 213 / モデル 1**。新しく社内アプリを作るときの出発点。

- 品目マスタの CRUD（一覧・登録・編集・無効化）が動く最小構成
- 検証、エラー表示、ソフトデリート、テーマ切り替えが**最初から入っている**
- これをコピーして中身を差し替えれば、規約に沿ったアプリがすぐ作れます

### 4. equipment-app — 備品管理
`pnpm dev:equipment` → http://localhost:3003

**画面 79 / API 213 / モデル 2**。小さな業務アプリの実例。

- 備品の登録、貸出、返却、貸出中一覧
- 簡易ログイン（管理パスワード）付き。**本番では秘密値の設定が必須**（未設定だと起動しません）
- 「1 機能に絞った小さなアプリ」の作り方の見本

### 5. platform-portal — 基盤ポータル
`pnpm dev:portal` → http://localhost:3005

**画面 1**。基盤そのもののカタログ。開発者が「何があるか」を探す場所。

- 107 パッケージを検索・カテゴリ絞り込み
- ADR（設計判断の記録）、ヘルス（健康診断）、Advisor（重複検出）、ER 図・依存グラフ
- テーマ切り替え付き（基盤ポータル自身がテーマ機構の実例）

## demos/ — 基盤の使い方を示すデモ（1個）

すべて **DB・外部サービス不要**（メモリ実装で代替）です。ただし 2 種類あります。

- **アプリ型（1つ）**: `showcase` のみ。`pnpm dev:demos`（:3001）でブラウザで動かせます
- **統合デモサイト（1）**: 起動しません。**ソースを読んで実装の参考にする**もので、部品や結線ロジックだけを置いています

AI に `find_examples("請求書")` と聞けば該当デモが返ります（[基盤カタログ MCP](ai/mcp-catalog.md)）。

### 総合デモ

| デモ | 内容 |
|---|---|
| **showcase** | 主要パッケージを 30 画面で体験（`pnpm dev:demos` → :3001）。フォーム、UI 部品、グラフ、暗号化、権限、Bluetooth、WebHID、リアルタイム、**テーマ機構**など |
| **app** | ログイン → ダッシュボード → 一覧 → フォーム → アップロードを 1 つに束ね、権限で出し分ける |

### 業務フロー

| デモ | 内容 |
|---|---|
| **procure-to-pay** | B2B の商流（見積→発注→検収→請求→支払）をロジックだけで通す |
| **attendance-to-ledger** | 勤怠集計 → 給与計算 → 明細 → 仕訳起票までを一本で辿る |
| **blueprint-flow** | 経費申請の手順（提出→承認→支払、差戻し）を宣言的に定義 |
| **blueprint-workflow** | 手順は blueprint、承認は workflow の**金額ルーティング + 多段承認**に委譲 |
| **accounting-sync** | 仕訳を freee へ送る結線例（外部 SaaS 連携の作り方） |

### 画面・UI

| デモ | 内容 |

## 使い分けの考え方

- **新しいアプリを作る** → `crud-template` をコピー
- **基盤に何があるか探す** → `platform-portal`（:3005）または `pnpm site` でリファレンスサイト
- **使い方を知りたい** → `demos/showcase`（:3001）で動かす、または該当デモの README
- **業務を回す** → `internal-app`（:3000）

すべてのアプリは同じ基盤の上に載っているので、**ある画面で使った部品は他でもそのまま使えます**。
