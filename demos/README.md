# demos — 統合デモサイト

**1 サイトだけ**置いています(`showcase`)。AWS Amplify にデプロイするのはこれ 1 つです。

| 種別 | 数 | 内容 |
|---|---|---|
| **統合デモサイト** | **1** | `showcase`(65 デモ・DB 不要) |

## なぜ 1 つなのか

以前は 27 フォルダありました(起動する 1 つ + 起動しない部品 26)。
**デプロイ対象が増える**ため、1 サイトに集約しました。

## 中身

```
showcase(:3001)
├── 基盤デモ(31)   @platform/* の使い方
├── アプリデモ(5)   apps/ の画面をモックで再現(実物ではない)
└── 使用例(9)      画面を持たないデモをコードで見せる
```

- **メニューで区分を分ける**ので、1 サイトでも別物として映ります。
- **DB を持ちません**(すべてメモリ・モックデータ)。Amplify 単体で完結し、RDS は不要です。

## 起動

```bash
pnpm dev:demos     # http://localhost:3001
```

## Amplify へのデプロイ

`showcase/amplify.yml` を使います。Amplify のコンソールで:

1. モノレポを選び、**App root を `demos/showcase`** に設定
2. ビルド設定は `amplify.yml` が読まれます
3. **バックエンド(RDS 等)は不要**

## 実物のアプリはどこか

`apps/` にあります。デモサイトの「アプリデモ」は**モックでの再現**で、実物ではありません。

| 実物 | 画面 | DB |
|---|---|---|
| `apps/internal-app` | 79 | 必要 |
| `apps/equipment-app` | 1 | 必要 |
| `apps/public-site` | 10 | 不要 |
| `apps/crud-template` | 1 | 必要 |
| `apps/platform-portal` | 1 | 不要 |

## 使用例のソース

`showcase/src/examples/` にあります。画面はそのソースを読んで表示するので、
**コードと表示が食い違いません**。
